Capítulo 2. Descripción GUI utilizados

Aquest capítol mostra les característiques de cada element utilitzat a l'alicació

Nota

Este capítulo muestra las características de cada elemento utilizado en la aplicación.

ElementoTipoDescripciónPropiedades principalesCaptura

Label

Control visual

Proporciona información en tiempo de ejecución o texto descriptivo para un control.

  • Text: Texto asociado al control

  • Font: Permite cambiar el tamaño, tipo de letra, color, etc... de la fuente.

TextBox

Control visual

Campo de texto que nos permite introducir datos para posteriormente tratarlos.

  • Multiline: Controla si el texto en el control de edición puede abarcar más de una lista

  • Font: Permite cambiar el tamaño, tipo de letra, color, etc... de la fuente.

Rich TextBox

Control visual

Proporciona una entrada de texto y características de edición avanzadas, como el formato de parráfo y de caracteres.

  • MaxLength: Especifica el número máximo de caracteres que se pueden introducir en el control de edición.

  • Font: Permite cambiar el tamaño, tipo de letra, color, etc... de la fuente.

Radio Button

Control visual

Permite al usuario seleccionar una única opción de entre un grupo de opciones cuando están emparejadas con otros RadioButtons.

  • Checked: Indica si el botón de radio está activado o no.

  • Text: Texto asociado al control

TabControl

Contenedor

Controla y muestra al usuario una colección relacionada de pestañas que pueden contener controles y componentes.

  • TabPages: Opción que permite agregar o quitar pestañas.

  • Alignment: Determina si las pestañas aparecen en la parte superior, inferior, izquierda o derecha del control.

Button

Control visual

Genera un evento cuando un usuario hace clic en él.

  • Text: Texto asociado al control

  • Enabled: Indica si el control está habilitado. Se puede habilitar/deshabilitar en función de un evento.

ComboBox

Control visual

Muestra un cuadro de texto editable con una lista desplegable de los valores permitidos.

  • Items: Elementos en el cuadro combinado.

  • MaxDropDownItems: Número máximo de entradas en la lista desplegable.

ToolStrip

Contenedor

Proporciona barras de herramientas y otros elementos de la interfaz de usuario que admiten varias opciones de apariencia.

  • Items: Colección de elementos que se van a mostrar en ToolStrip.

  • LayoutStyle: Especifica la orientación de diseño de ToolStrip.

ListView

Contenedor

Muestra una colección de elementos en una de cinco vistas diferentes.

  • Columns: Columnas mostradas en la vista de detalles.

  • LabelEdit: Permite que el usuario modifique las etiquetas de los elementos directamente.

Nota

Este capítulo explica los diferentes formularios de la aplicación y su uso.

Form1

Este Form, valida un usuario y contraseña para poder iniciar la aplicación. Una vez introducidos ambos campos, puedes acceder a la aplicación.

Form2

Este Form, es un formulario de entrada de datos. Los campos son:

  1. Nom: Campo tipo TextBox donde introducimos el Nombre del paciente

  2. Cognom: Campo tipo TextBox donde introducimos el apellido del paciente

  3. Sexe: Campo de tipo radioButton donde podemos seleccionar entre H (Hombre) y M (Mujer)

  4. Nacionalitat: Campo tipo TextBox donde introducimos la nacionalidad del paciente

Form3

Este Form,es un formulario que muestra el listado de todos los pacientes dados de alta.

Nota

El formulario tiene las siguientes opciones de filatrado:

  1. Filtrar por hombre (H)

  2. Filtrar por Mujer (M)

  3. Mostrar todos los registros(Tots). Opción por defecto

Form4

Este Form,es un formulario que muestra el listado de todos los pacientes. Si hacemos clic en el botón que representa a un paciente, obtenemos la información de su historial.

Detalle de opciones:

  1. Ver Historial de cada paciente

  2. Ir al formulario 3 para visualizar el listado completo en formato tabla (Botón Resum Pacients
